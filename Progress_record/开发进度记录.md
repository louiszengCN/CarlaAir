## 一、项目概述

**项目名称：** CarlaAir（CARLA-Air，空地一体联合仿真平台）

**目标：** 在 CARLA 城镇地图中飞行 AirSim 无人机，两套 API 在同一个 UE4 进程中同时工作。

**当前状态：** ✅ v0.1.6。在 v0.1.5 基础上增加：**自动交通生成**（启动即有30辆车+50个行人异步运行，无人机俯瞰即可看到熙熙攘攘的城市场景）、**H键帮助菜单优化**（紧凑单行双语、20pt粗体+阴影、MaxDesiredHeight防溢出）。`ASimWorldGameMode` 统一 GameMode，内置 FPS 无人机控制（WASD+鼠标），碰撞弹开+地面夹持物理效果，ROS2 适配验证（63 个话题），CARLA 所有功能（天气、车辆/行人生成、Traffic Manager、OpenDRIVE）+ AirSim 无人机飞行同时工作。提供轨迹录制/回放/视频录制脚本及15个v5 Demo脚本，支持空地一体 Demo 制作。

**技术架构：**
- 单个 UE4 Editor 进程同时运行 CARLA 和 AirSim
- GameMode：使用统一的 `ASimWorldGameMode`（继承自 `ACarlaGameModeBase`，同时集成 AirSim 引导逻辑）
- GameInstance：使用 CARLA 的 `CarlaGameInstance`（在 DefaultEngine.ini 中配置）
- AirSim 的 `ASimModeBase` 作为普通 Actor 由 GameMode 在 BeginPlay 中 Spawn
- CARLA Episode 由 `ACarlaGameModeBase` 构造函数创建，在 `BeginPlay()` 中初始化（通过继承自动完成）

**硬件环境：**
- GPU: NVIDIA RTX A4000, 16GB VRAM
- 系统: Linux (Ubuntu)

### 已解决的根本问题

**GameMode 冲突（核心问题）：** UE4 每个 World 只允许一个 GameMode。之前使用 AirSim 的 GameMode 导致 30+ 个 CARLA 功能不可用。

**解决方案：** 创建 `ASimWorldGameMode`（继承自 `ACarlaGameModeBase`）：
- 所有 CARLA 功能通过继承获得（天气、车辆生成、OpenDRIVE 等）
- AirSim 引导逻辑从 `ASimHUD` 移植到 GameMode 的 BeginPlay 中
- `ASimModeBase` 作为普通 Actor 被 Spawn，不再竞争 GameMode 槽位
- 构造函数中手动设置 `WeatherClass`（BP_Weather）和 `ActorFactories`（8 个工厂类）

## 五、已修复的关键 Bug

| Bug | 原因 | 修复方式 |
|-----|------|----------|
| `get_world()` 时 SIGSEGV 崩溃 | Episode 对象被 UE4 GC 回收 | 使用 `AddToRoot()` 防止 GC |
| 多个 RPC handler null 崩溃 | 16+ 个 handler 假设 CARLA GameMode 存在 | 添加 null guard 和 fallback |
| `CarlaLight::GetLocation()` 崩溃 | `GameMode->GetLMManager()` 在 null 上调用 | 添加 null check |
| `VK_ERROR_OUT_OF_DEVICE_MEMORY` | 运行 5 小时后 GPU 内存泄漏 | 启动参数添加 `-TexturePoolSize=2048` |
| 编辑器模式加载 Cooked 内容失败 | `CARLA_Latest.tar.gz` 包含 Shipping 资产 | 下载源码版内容（21GB） |
| `Missing weather class` | `WeatherClass` 为 UPROPERTY，需蓝图设置 | 构造函数中用 FClassFinder 加载 BP_Weather |
| `AUnrealLog` 重复定义 | AirSimGameMode.cpp 已定义该类 | 重命名为 `ASimWorldUnrealLog` |
| Spectator 缺失 (`unable to find spectator`) | `DefaultPawnClass=nullptr` 导致 PlayerController 无 Pawn，Episode 找不到 Spectator | 在 BeginPlay 中生成未 Possess 的 `ASpectatorPawn`，直接设置 `Episode->Spectator` 并注册到 `ActorDispatcher` |
| AirSim 无人机不动（设置 DefaultPawnClass 后） | PlayerController 上有 Possessed Pawn 会干扰 AirSim SimpleFlight 输入 | 必须保持 `DefaultPawnClass=nullptr`，Spectator 不能被 Possess |
| **CARLA 车辆穿地**（z 从 0 掉到 -30m） | AirSim `SimModeBase::BeginPlay()` 调用 `SetNewWorldOrigin()` 将 UE4 世界坐标原点移至 PlayerStart（z≈30m），导致 Landscape 碰撞几何体失效，车辆失去地面碰撞 | 注释掉 `SimModeBase.cpp:121` 的 `SetNewWorldOrigin()` 调用，保留 `NedTransform` 使用原始 `player_start_transform` 作为 NED 坐标原点 |
| `fly_drone_city.py` IOLoop 崩溃 | pynput 键盘监听线程中直接调用 AirSim API，与主线程的 tornado IOLoop 冲突 | 回调只设置 `pending_actions` 标志，所有 AirSim API 调用移到主线程循环中处理 |
| **传感器并发崩溃** (4+ sensors ~4min SIGABRT) | 1) IMU 传感器 `ComputeGyroscope()` 中 `check(GetOwner() != nullptr)` 致命断言：父车辆被销毁后 Owner 变 null，触发 SIGSEGV；2) GPU readback 操作无上限累积 | 1) `InertialMeasurementUnit.cpp`: 将 `check` 替换为 null guard + 零向量返回；`PostPhysTick` 入口添加 Owner/PendingKill 安全检查；2) `PixelReader.cpp`: 添加 `GPendingReadbacks` 原子计数器，限制并发 readback ≤ 16 |
| **FPSYaw_ 初始化为 0** (W键方向错误) | `FPSYaw_` 初始化为 `0.0f`（NED 正北），但无人机可能朝其他方向 | `UpdateFPSControl()` 中在 `bDroneReady_` 后读取无人机实际 Yaw 初始化 `FPSYaw_` |
| **按 `-` 键黑屏** | `EKeys::Hyphen` 与 UE4 引擎内置的 `EngineShowFlags::Rendering` 切换冲突 | 速度调节键改为 `EKeys::RightBracket`（`]`）/`EKeys::LeftBracket`（`[`） |

## 七、一键启动 — carlaAir.sh

### 基本用法

```bash
cd /mnt/data1/tianle/carla_source

# 默认启动（Town10HD, 1280x720, 低画质）
./carlaAir.sh

# 指定地图
./carlaAir.sh Town03

# 自定义分辨率
./carlaAir.sh Town10HD 1920 1080

# 前台运行（可看到所有日志）
./carlaAir.sh --fg

# 停止运行
./carlaAir.sh --kill

# 查看实时日志
./carlaAir.sh --log

# 查看帮助
./carlaAir.sh --help
```

### 启动流程

1. 脚本检查是否有已运行的实例
2. 设置 DISPLAY 和 Vulkan 环境变量
3. 后台启动 UE4Editor
4. 自动轮询 CARLA (2000) 和 AirSim (41451) 端口
5. 两个端口都就绪后打印 "Ready!" 并提供测试命令

### 启动参数说明

| 参数 | 说明 |
|------|------|
| `-game` | 以游戏模式运行（非编辑器 UI） |
| `-windowed -ResX -ResY` | 窗口化运行及分辨率 |
| `-carla-rpc-port=2000` | CARLA API 监听端口 |
| `-quality-level=Low` | 低画质，节省 GPU 显存 |
| `-TexturePoolSize=2048` | 限制纹理池大小，防止显存泄漏 |
| `-unattended -nosound` | 无需用户交互，禁用声音 |

**跟随策略**：使用相对偏移法
1. 记录车辆初始 CARLA 坐标 `(car_x0, car_y0)` 和无人机初始 AirSim 坐标 `(drone_x0, drone_y0)`
2. 每帧计算车辆位移 `dx = car_x - car_x0`, `dy = car_y - car_y0`
3. 目标无人机位置 = `(drone_x0 + dx, drone_y0 + dy, -altitude)`

## 十四、车辆穿地 Bug 修复 (2026-03-11)

### 问题描述

在 CarlaAir 中通过 CARLA API 生成的地面车辆会立即穿过地面，掉落约 30 米（从 z≈0.6 到 z≈-29.6）。而在原版 CARLA（无 AirSim 插件）中，相同操作完全正常（z≈0.0）。此 Bug 影响 Editor 和 Shipping 两种构建。

### 排查过程

1. **对比测试**：原版 CARLA（`/home/tianle/carla/CARLA_Latest/`）加载 Town10HD_Opt，车辆 z≈0.0 正常；CarlaAir 加载 Town10HD，车辆 z≈-29.6 穿地
2. **排除地图因素**：CarlaAir 使用 Town10HD_Opt 仍然穿地 → 不是地图问题
3. **确认 AirSim 插件为根因**：唯一区别是 AirSim 插件的存在

### 根本原因

**文件**: `Plugins/AirSim/Source/SimMode/SimModeBase.cpp`，第 121 行

```cpp
// BeginPlay() 中：
this->GetWorld()->SetNewWorldOrigin(FIntVector(player_loc) + this->GetWorld()->OriginLocation);
```

**机制**：
1. `ASimModeBase::BeginPlay()` 获取 PlayerStart/ViewTarget 的位置（Town10HD 中 PlayerStart 在海岸，z≈30m）
2. 调用 `SetNewWorldOrigin()` 将 UE4 世界坐标原点移到该位置
3. 此操作导致 **Landscape/道路碰撞几何体失效** — UE4 的 PhysX 不正确地更新了地面碰撞
4. CARLA 车辆生成在 z=0.6（来自 OpenDRIVE 数据），但地面碰撞已不存在
5. 重力将车辆拉到 z≈-30（穿过不可见的地面）

**为什么无人机不受影响**：AirSim 的 Multirotor 使用自定义 `FastPhysicsEngine`，不依赖 UE4 Landscape 碰撞；而 CARLA 车辆使用 UE4 原生 PhysX，依赖 Landscape 碰撞体。

### 修复方案

注释掉 `SetNewWorldOrigin()` 调用，保留 `NedTransform` 使用原始 `player_start_transform` 作为 NED 坐标原点：

```cpp
// 修复前（SimModeBase.cpp:117-125）:
player_start_transform = fpv_pawn->GetActorTransform();
player_loc = player_start_transform.GetLocation();
this->GetWorld()->SetNewWorldOrigin(FIntVector(player_loc) + this->GetWorld()->OriginLocation);
player_start_transform = fpv_pawn->GetActorTransform();
global_ned_transform_.reset(new NedTransform(player_start_transform, ...));

// 修复后:
player_start_transform = fpv_pawn->GetActorTransform();
player_loc = player_start_transform.GetLocation();
// NOTE: Do NOT call SetNewWorldOrigin() — it breaks CARLA's landscape collision
global_ned_transform_.reset(new NedTransform(player_start_transform, ...));
```

### 验证结果

| 测试项 | 修复前 | 修复后 |
|--------|--------|--------|
| 5 辆车辆 z 位置（5s后）| z≈-27.6（穿地） | z≈0.00（正常） |
| 30 辆车辆穿地数 | 30/30 全部穿地 | 0/30 穿地 |
| 车辆 autopilot 行驶 | 无法行驶（在地下） | 18/30 正常行驶 |
| 原版 `generate_traffic.py` | 无法使用 | ✅ 完美兼容 |
| AirSim 无人机飞行 | 正常 | 正常（不受影响） |
| AirSim 相机拍照 | 正常 | 正常（不受影响） |
| CARLA + AirSim 同时运行 | 车辆穿地 | ✅ 30 辆车 + 无人机完美协同 |

### 坐标系影响

修复后坐标系映射更新：

| | 修复前 | 修复后 |
|---|---|---|
| NED 原点 | UE4 (0,0,0)（原点被移到 PlayerStart） | PlayerStart 实际位置（海岸附近） |

## 二十一、v0.1.2 — 内置 FPS 无人机控制 + 启动脚本改进 (2026-03-13)

### 概述

v0.1.2 包含三类改进：
1. **C++ FPS 无人机控制** — 在 UE4 二进制中内置键盘/鼠标无人机控制（无需 Python）
2. **CarlaAir.sh 改进** — Vulkan 自动检测（支持 NVIDIA/AMD/Intel）、默认 Epic 画质、OpenGL 回退
3. **Python 坐标修复** — `drone_path_recorder.py` / `drone_path_player.py` 使用 CARLA actor 真实坐标

### 1. C++ FPS 无人机控制

#### 架构

```
            ┌─────────────────────────────┐
            │      Tick() (Game Thread)    │
            │  - 读取键盘 (WASD/QE)        │
            │  - 读取鼠标增量 (yaw/pitch)   │
            │  - 计算世界坐标系速度          │
            │  - 写入共享状态               │
            │  - 更新 Spectator 摄像机      │
            └──────────┬──────────────────┘
                       │ FCriticalSection
            ┌──────────▼──────────────────┐
            │  FDroneControlWorker (FRunnable)│
            │  - 读取共享速度/偏航           │
            │  - moveByVelocity() [阻塞]    │
            │  - ~20Hz 控制循环              │
            └───────────────────────────────┘
```

**为什么需要后台线程**：AirSim 的 `moveByVelocity()` 是阻塞调用（内部使用 `waitForFunction`），不能在游戏线程 `Tick()` 中调用。后台 FRunnable 线程通过 `FCriticalSection` 与游戏线程共享速度/偏航状态，实现流畅的 FPS 风格控制。

#### 操控按键（最终方案）

| 按键 | 功能 | 实现 |
|------|------|------|
| W/S | 前进/后退 | `PC->IsInputKeyDown()` → 基于 yaw 分解的水平面世界坐标速度 |
| A/D | 左/右平移 | 同上 |
| Space | 上升 | 垂直速度 (NED z 轴负方向 = 上升) |
| Left Shift | 下降 | 垂直速度 (NED z 轴正方向 = 下降) |
| 鼠标 X | 偏航 (Yaw) | `PC->GetInputMouseDelta()` → 累加偏航角，灵敏度 0.35 |
| V | 切换视角 | 第一人称 ↔ 第三人称（无人机后方 8m + 上方 3.5m） |
| N | 切换天气 | 循环 6 种预设（晴天/多云/雨天/日落/夜晚/沙尘暴） |
| M | 切换地图 | 通过 `Episode->LoadNewEpisode()` 切换（重启关卡） |
| +/- | 调速 | 调节无人机速度 (2~30 m/s) |
| Tab | 鼠标捕获 | 切换游戏输入 / 自由光标 |
| R, ;, F1, T, 1-3, 0 | AirSim 原有绑定 | 保持不变 |

> **控制方案演进**: 经多轮测试迭代，最终确定鼠标只控制 yaw（不控制 pitch），WASD 纯水平面移动，Space/Shift 纯垂直升降。此方案最直观稳定，避免了 pitch 影响飞行方向带来的操控困难。

#### 修改文件

| 文件 | 行数 | 变更 |
|------|------|------|
| `SimWorldGameMode.h` | 127 行 | +50 行：FPS 状态成员、线程成员、共享状态、天气/地图数组、输入处理方法声明 |
| `SimWorldGameMode.cpp` | 981 行 | +250 行：FDroneControlWorker 类、SetupFPSControl、UpdateFPSControl、UpdateCameraFollow、6 个输入处理函数、EndPlay 线程清理 |

#### FDroneControlWorker 实现要点

- **延迟激活**：启动时不调用 `enableApiControl`/`armDisarm`，让 Python API 可自由控制无人机；首次检测到键盘输入时才激活 FPS 控制
- **控制循环**：读取共享状态 → 始终调用 `moveByVelocity(vx, vy, vz, 0.1, MaxDegreeOfFreedom, YawMode(false, yaw))`，即使悬停也发送 `moveByVelocity(0,0,0, yaw)` 以保证 yaw 同步
- **错误处理**：所有 AirSim API 调用包裹在 `try-catch` 中，防止线程崩溃
- **关闭**：`land()` → `armDisarm(false)` → `enableApiControl(false)`

> **为什么不用 `hover()`**: `hover()` 不接受 yaw 参数，悬停时 yaw 不更新。始终使用 `moveByVelocity(0,0,0, yaw)` 确保鼠标转向在任何时候都生效。

### 2. CarlaAir.sh 改进

#### 2a. Vulkan 自动检测

替换硬编码 `nvidia_icd.json` 为自动扫描函数：

```bash
detect_vulkan_icd() {
    local ICD_DIRS="/usr/share/vulkan/icd.d /etc/vulkan/icd.d /usr/local/share/vulkan/icd.d"
    local ICD_FILES=""
    for dir in ${ICD_DIRS}; do
        if [ -d "${dir}" ]; then
            for f in "${dir}"/*.json; do
                [ -f "$f" ] && ICD_FILES="${ICD_FILES:+${ICD_FILES}:}$f"
            done
        fi
    done
    [ -n "${ICD_FILES}" ] && export VK_ICD_FILENAMES="${ICD_FILES}"
}
```

### 3. Python 坐标修复

#### 问题

`drone_path_recorder.py` 使用 NED 坐标偏移量估算 CARLA 坐标，但偏移量不准确。

**根本原因**: AirSim 无人机 pawn 未注册到 CARLA 的 `ActorDispatcher`，导致 `world.get_actors()` 找不到无人机 actor。
`calibrate_offset()` 只能回退到 CARLA spectator 位置，但 spectator 初始位于 UE4 原点 (0,0,0)，而非 PlayerStart（NED 原点），偏移量因此错误。

#### 修复（C++ + Python）

**C++ 修复**: `SimWorldGameMode.cpp` — 在 `Tick()` 中一次性注册无人机 pawn 到 CARLA ActorDispatcher:

```cpp
// SimWorldGameMode::Tick() — 无人机注册
auto* DroneSimApi = static_cast<PawnSimApi*>(SimMode_->getVehicleSimApi());
APawn* DronePawn = DroneSimApi->getPawn();
FActorDescription Description;
Description.Id = TEXT("airsim.drone");
Description.Class = DronePawn->GetClass();
Episode->ActorDispatcher->RegisterActor(*DronePawn, Description);
```

注册后，Python 脚本通过 `world.get_actors()` 可找到 `type_id='airsim.drone'` 的 actor，获取精确 CARLA 坐标：

```python
# 查找无人机 actor（支持重试，等待首次 Tick 注册完成）
drone_actor = find_carla_drone(world, retries=10, interval=0.5)

# 每帧获取精确 CARLA 坐标
t = drone_actor.get_transform()
carla_x, carla_y, carla_z = t.location.x, t.location.y, t.location.z
```

**Python 修复**: `drone_path_recorder.py` / `drone_path_player.py` — `find_carla_drone()` 增加重试机制，`calibrate_offset()` 移除不可靠的 spectator 回退。

### 3b. Sweep 碰撞修复（关键 Bug）

#### 问题

在 CARLA 地图中，无人机能悬停但**不能飞行**：`moveToPosition`、`moveByVelocity` 等命令发出后无人机仍然坠落。

**根本原因链**:
1. `MultirotorPawnSimApi::updateRendering()` 调用 `PawnSimApi::setPose(last_phys_pose_, false)`
2. `false` = 不忽略碰撞 → 使用 UE4 的 sweep 碰撞检测模式
3. `SetActorLocationAndRotation(pos, orient, true)` — sweep 在 CARLA 地图中检测到**虚假碰撞**（可能与景观网格或 PlayerStart 碰撞体冲突）
4. `onCollision` 触发，设置 `collision_info.has_collided=true` 并更新时间戳
5. `updateRenderedState()` 将碰撞信息传递到物理体
6. `FastPhysicsEngine` 检测到新碰撞时间戳 → 调用 `getNextKinematicsOnCollision()`
7. 碰撞响应**置零所有力/力矩** (`next_wrench = Wrench::zero()`)，**覆盖位置**到碰撞点
8. 悬停时：pawn 不移动 → 无新 sweep → 碰撞时间戳不变 → 仅触发一次 → **稳定**
9. 飞行时：pawn 移动 → 新 sweep → 新碰撞 → **每个物理 tick 都触发碰撞响应** → 无人机坠落

#### 修复

**文件**: `MultirotorPawnSimApi.cpp` 第 110 行
```cpp
// 修复前:
PawnSimApi::setPose(last_phys_pose_, false);  // sweep 模式

// 修复后:
PawnSimApi::setPose(last_phys_pose_, true);   // teleport 模式（避免虚假 sweep 碰撞）
```

#### 补充修复：地面夹持 + 碰撞弹开 (teleport + sphere overlap)

Teleport 模式跳过了 UE4 的 sweep 碰撞检测，导致无人机穿模（穿过建筑/地面无碰撞）。

**解决方案**: 在 `updateRendering()` 中 teleport 之后，实现两层手动碰撞检测：

**1. 地面夹持 (Ground Clamp)**: 防止无人机穿过地面
```cpp
// Line trace: 从无人机上方 100m 到下方 5m
pawn->GetWorld()->LineTraceSingleByChannel(
    GroundHit,
    DronePos + FVector(0, 0, 10000.0f),
    DronePos - FVector(0, 0, 500.0f),
    ECC_WorldStatic, GroundParams);
if (DronePos.Z < GroundHit.ImpactPoint.Z)
    DronePos.Z = GroundHit.ImpactPoint.Z + 5.0f;  // 推到地面上方 5cm
```

**2. 球形碰撞弹开 (Sphere Overlap Bounce)**: 检测建筑/障碍物并弹开无人机
```cpp
// 60cm 半径球形碰撞检测
pawn->GetWorld()->OverlapMultiByChannel(
    Overlaps, DronePos, FQuat::Identity,
    ECC_WorldStatic, FCollisionShape::MakeSphere(60.0f), QueryParams);
// 检测到重叠时，计算推离方向，将无人机推离障碍物表面 80cm
FVector Away = DronePos - ClosestPoint;
FVector BouncePos = DronePos + Away.GetSafeNormal() * 80.0f;
pawn->SetActorLocation(BouncePos, false, nullptr, ETeleportType::TeleportPhysics);
```
- **弹开而非 reset**: 不调用 `NotifyHit()`（会触发 AirSim 物理引擎碰撞响应，置零所有力并 reset 位置），仅通过 `SetActorLocation` 推离
- 过滤 `APawn` 类型（排除 SpectatorPawn 等摄像机 Actor 的误报）
- 推离方向通过 `GetClosestPointOnCollision` 计算最近表面点

**文件**: `MultirotorPawnSimApi.cpp` 第 112-169 行

> **设计决策**: 早期版本使用 `NotifyHit()` 触发碰撞信号链，但这会导致 `FastPhysicsEngine::getNextKinematicsOnCollision()` 置零所有力/力矩并重置无人机位置（类似坠毁重置），用户体验极差。弹开方案保留碰撞物理感的同时不中断飞行。

- **AirSim API 兼容**：碰撞信息写入 `state_.collision_info`，Python `simGetCollisionInfo()` 可查询

### 核心技术贡献（论文可用）

1. **统一GameMode架构 (ASimWorldGameMode)**：解决UE4单GameMode约束，采用单继承+组合模式，继承ACarlaGameModeBase获得CARLA全部功能，AirSim的ASimModeBase作为普通Actor挂载。仅修改CARLA源码2个文件（CarlaGameModeBase.h: private→protected; CarlaEpisode.h: 添加friend class）。

2. **双API服务器共存**：CARLA rpclib (port 2000, 三端口架构: 2000/2001/2002) + AirSim rpclib (port 41451) 在同一UE4进程中运行，共享渲染管线和物理引擎。用户现有Python脚本零修改即可使用。

3. **Spectator Pawn解耦**：AirSim要求DefaultPawnClass=nullptr（否则SimpleFlight输入失效），但CARLA需要Episode->Spectator。解决方案：生成ASpectatorPawn但不Possess，通过friend class直接设置Episode->Spectator。

4. **内置FPS交互控制系统**：C++实现的键盘/鼠标无人机控制，FDroneControlWorker后台线程(~20Hz)通过FCriticalSection与游戏线程共享状态。经5轮控制方案迭代+4轮碰撞检测迭代确定最终方案：鼠标仅Yaw + WASD水平 + Space/Shift垂直。

5. **碰撞物理系统**：Teleport模式替代UE4 Sweep（避免CARLA地图虚假碰撞），12方向LineTrace碰撞检测（8水平+4斜下，150cm检测距离，180cm弹开距离），地面夹持（Line Trace），P键物理/穿越模式切换。

6. **精确坐标对齐**：禁用AirSim的SetNewWorldOrigin()（此调用破坏CARLA Landscape碰撞），使CARLA↔AirSim坐标精确对齐（0.0000m误差）。无人机作为airsim.drone注册到CARLA ActorDispatcher，Python API可获取精确CARLA坐标。

7. **传感器稳定性修复**：IMU GetOwner()空指针致命断言替换为null guard + GPU readback原子计数器限制并发≤16。修复后6传感器10min稳定运行（28986/24866/59053帧）。

8. **Demo录制管线**：30个Demo视频（1920×1080，总计2.1GB），标准四格构图（Drone FPV / Drone 3rd / Vehicle FPV / Vehicle 3rd），7个不同地图。发现并解决CARLA sync mode + AirSim simGetImages渲染死锁问题。

## v0.1.6 更新日志 (2026-03-18)

### 核心新功能：自动交通生成

**需求**：启动 CarlaAir 后，城市中自动就有车辆和行人在运动。用 WASD 飞无人机时，俯瞰下方就能看到熙熙攘攘的城市场景，真正体现空地一体的概念。

**实现方案**：

1. **`auto_traffic.py`** — 异步交通生成器守护进程
   - 路径：`/mnt/data1/tianle/carla_source/auto_traffic.py`
   - 在 CARLA 异步模式下运行，不影响 FPS 无人机控制
   - 默认生成 30 辆车（4轮汽车，autopilot + Traffic Manager）+ 50 个行人（AI Walker Controller）
   - 行人每 30 秒自动更换目的地，防止站立不动
   - 支持 SIGTERM/SIGINT 信号，退出时自动清理所有生成的 actor
   - 可通过参数自定义：`--vehicles 30 --walkers 50 --port 2000`

2. **`carlaAir.sh` 自动集成**
   - CARLA + AirSim 服务器就绪后，自动后台启动 `auto_traffic.py`
   - 等待 8 秒确认交通生成成功
   - `./carlaAir.sh --kill` 时同时停止交通生成器
   - 交通日志输出到 `/tmp/carla_traffic.log`

3. **效果验证**
   - Town10HD：30/30 车辆 + 47/50 行人成功生成
   - 鸟瞰视角确认：十字路口有车辆行驶，人行道有行人走动
   - 无人机飞行时不受交通生成影响（异步模式独立运行）

### 碰撞物理优化（v0.1.6 release 最终修复）

旧问题：撞到建筑物时瞬移 1.8m（不自然）、贴墙抖动、被向上抬升。

新碰撞系统：
- **速度反射弹回**：计算碰撞面法线，反射无人机速度，产生自然弹开效果
- **二次衰减**：弹回速度随时间二次递减（0.4秒），起始快末尾慢
- **弹回期间屏蔽输入**：SimWorldGameMode 检测 `isBouncing()` 状态，强制 hover 防止玩家立即飞回
- **冷却机制**：0.3秒冷却防止连续碰撞抖动
- **能量损失**：反弹系数 0.6，每次碰撞损失 40% 动能
- **最小弹回速度**：即使低速碰撞也有 3m/s 的弹回力度
- **真实法线**：使用 `Hit.ImpactNormal`（而非射线方向取反），碰撞方向更准确

文件变更：
- `MultirotorPawnSimApi.h` — 新增弹回状态变量（BounceVelocity, TimeRemaining, Cooldown）
- `MultirotorPawnSimApi.cpp` — 重写碰撞检测+弹回逻辑
- `SimWorldGameMode.cpp` — 弹回期间屏蔽 FPS 控制输入
